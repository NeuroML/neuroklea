## Key decisions

- **Node labels ARE LangGraph node names**: no separate metadata dict or
  lookup table. `run_graph_astream_events()` receives them natively via
  `langgraph_node` metadata (LLM nodes) and custom events (non-LLM).
- **`_CustomChannelEnabler` transformer required**: LangGraph v3 only emits
  channels that registered transformers declare via `required_stream_modes`.
  This no-op transformer exists solely to enable `custom` for non-LLM progress.
- **Token streaming skipped for answer text**: the answer flows through as
  JSON schema fragments, post-processed by a non-LLM node. Frontend will
  simulate typewriter from the complete `message_for_user` string.
- **Debug panel approach**: server streams all events unfiltered; frontend
  dispatches progress labels to main UI and raw tokens to a collapsible debug
  sidebar.

## Errors and Lessons

- `StreamTransformer` requires `init()` -- discovered by reading LangGraph
  source; would have crashed at runtime without it.
- `langgraph_node` metadata only surfaces in `messages` channel, confirmed
  by reading `_algo.py` and `_messages.py`.
- `dict` vs `RunnableConfig` type errors were fixable with one import and
  annotation across 5 lines.
