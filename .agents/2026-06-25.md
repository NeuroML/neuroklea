# Session: 2026-06-25

**Authoring agent:** opencode

## Objective
Refactored vector-store builder into modular subcommands with chunk caching, per-file metadata-map templates, and cleaned up config model hierarchy.

## Completed
- **Restructured `klea-vs-create`**: split from single `create` command into `build`, `chunk`, `store` subcommands using Typer groups. `build` is the full pipeline; `chunk` converts+caches+generates heading template without writing to store; `store` reads cache and writes to store.
- **Extracted `chunk_all()` / `store_all()` public methods** on `VSBuilder`, removing the `chunk_only` mode flag entirely. `build()` is a thin convenience wrapping both.
- **Added chunk cache** (`<source_dir>/.klea-cache/<xxh64_hash>.pkl`). Cached on first conversion; subsequent runs load from pickle instead of re-converting with Docling.
- **Per-file metadata-map templates**: `chunk` now writes `{file_name: {"DEFAULT": {}, "heading chain": {}}}` instead of a flat heading list. `_resolve_metadata` updated to match by file name then heading chain (no backwards compat with flat format).
- **Enhanced logging**: added INFO log lines for metadata map loading, embedding model init, store openings, Docling conversion steps, cache writes. JSON parse errors now point to the specific file.
- **Skipped `.klea-cache/` in file discovery** to avoid "unsupported file" warnings for `.pkl` files.
- **`serialize_vs_retrieval`**: updated for Docling's new `headings` list format with relevance score and compact metadata dump.
- **Config model consolidation**: `PerDomainConfig` subclassed across `utils_pkg`/`rag_pkg`; `VectorStoresConfig` slimmed to only store topology; `default_k`/`k_max`/`embedding_model` moved to constructor params of `VSRetriever` and `BaseLangGraph`.
- **Updated `klea-vs-create --help`** with metadata-map format examples.
- **`:` -> `_`** in cache filenames for filesystem safety.
- **Progress counter** in chunking logs (`file.pdf (3/12)`).

## Errors and Lessons
- Docling's PDF layout inference is slow but idempotent  ---  caching is essential for re-runs.
- The old cache files used `xxh64:abcdef.pkl` (with `:`) but the new code looks for `xxh64_abcdef.pkl`  ---  manual `rename ':' '_' .klea-cache/*` needed for existing caches.
- Quadro P1000 GPU incompatible with PyTorch 2.x  ---  `CUDA_VISIBLE_DEVICES=""` needed to force CPU fallback.
- Typer uses `...` (Ellipsis) as a sentinel for required options, checked via `is ...`.
